The input layer applies convolution on the image using 32 filters.
32 is a large number that allows increasing the number of filters in next layers.
To avoid overfit and to spread info,each Conv2D is followd by a MaxPooling layer to reduce the image resolution by a half.
Due to this,the number of filters doubles to account for the loss in resolution.
Since the model converged to 0.88 val_accuracy, I used normalization between each layer to make the data more general and increase the accuracy.
The last dense layer has 100 nodes since with 120 it starts to overfit and with less it was converging to 0.8.
Because the model started to converge to 0.9,I added dropout before and after the layer with 100 nodes in order to break any patterns.
Picked batch size of 100 to increase learning rate.

The model confuses classes 3, 4, and 7 with class 1 and classes 5, 8 with class 3. Also, class 9 is confused with 2.
I analyzed digit 0.
From the confusion matrix, it can be seen that the model has a noticeable problem differentiating 0 from 1.
Some 0 digits are written using the old digital clock font, making it hard to diferentiate 0 from 11.
Some of them have the digit 0 written too wide while other have it too narrow, making it hard to diferentiate 0 from 11.
The other digits which were missclassified as 0, seem to have more than one digit in the image. Such images should not be in the dataset, since the model has to pick a digit based on no criteria.
Other digits seem to be of poor quality, having lots of background noise.
