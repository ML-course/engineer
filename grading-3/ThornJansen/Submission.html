Note that the value behind missclassified_samples can be adjusted to see different missclasifications for the number 4. What can be noted is that a lot of misclassifications have to do with noise. The pictures are very unclear. It seems like the network struggles with pictures which have a very unclear outline of the number. There are also a number of missclasifications where there is a (partial) second number on the image. Misclassifications also happen when the background is very similar to the number. Although there are not to many cases of this last problem. Lastly note that also some answers are incorrect for certain samples. Due to the low quality images it confuses numbers which vertical lines e.g 1 and 7. and numbers which a round shape e.g 6,8,9 more often.
